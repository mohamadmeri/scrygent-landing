Three passes. Zero guesses.

Every query moves through the same disciplined pipeline. Nothing about the answer is left to chance, and nothing about the plan is left to a script the model wrote itself.

01 / Read

It reads before it answers.

Every dataset is profiled first. Schema, types, null rates, representative rows. The planner never guesses at what's in your file; it is told.

02 / Plan

It plans, it doesn't script.

The model chooses what to compute. Filters, aggregations, regressions. It emits them as a strict, typed instruction. Not a Python script. Not a sandbox.

03 / Compute

It proves, it doesn't hallucinate.

Every number in the final report traces back to deterministic code. If a step can't be expressed validly, the run stops. It never guesses instead.

Why not just let it write Python? Because code generation fails quietly.

Most "chat with your data" tools let the model write and execute arbitrary Python in a loop. It works, until it doesn't: a hallucinated column name, a silently wrong aggregation, a plot that never matches the numbers next to it.

Scrygent's planner never touches your data. It emits a strict, validated instruction (the IR shown here), and a handwritten Python engine is the only thing allowed to compute anything. The LLM decides what; the engine decides how.
